Why an emulator

Robot time is scarce and mistakes on real hardware are expensive. This endpoint gives you the API surface with none of the blast radius.

Every tracked version

Each MiR software generation is mounted at its own prefix — /<version>/api/v2.0.0/… — so you can diff behavior across releases from one URL and catch upgrade breakage early. Pick the version in the console's software version panel.

Real behavior, real auth

Missions execute over simulated time: the queue drains FIFO with real ordered/started/finished timestamps, battery drains, position moves, PLC registers hold writes, and unknown mission ids are rejected with 400 like a factory robot. Auth mirrors one too — Basic BASE64(user:SHA-256(password)), default distributor/distributor.

Made for hardening

Deterministic responses generated from the official API definitions, documented error paths (400/404/409) you can trigger on purpose, and per-session virtual robots — send an X-MiR-Session header and you get your own isolated fleet state.

⚠ Shared demo unit

This console talks to a live, public emulator of the Mobile Industrial Robots REST API running on AWS Lambda. Each tracked MiR software generation is mounted at its own path prefix — /<version>/api/v2.0.0/… — so you can diff robot behavior across releases from one URL. State (mission queue, PLC registers, status writes) is in-memory per runtime instance and resets on cold start. Auth mirrors a factory robot: Basic BASE64(user:SHA-256(password)), default account distributor / distributor. Once your integration works against the demo, switch the endpoint to a real MiR robot's IP or hostname on your network — same console, same requests.

Start in thirty seconds

  

    

      

        1 — From your terminal. The demo speaks plain HTTPS; the only trick is MiR's hashed-password Basic auth.
        A robot only queues missions it knows, so list them first — exactly like a real one:

TOKEN=$(printf 'distributor:%s' "$(printf 'distributor' | shasum -a 256 | cut -d' ' -f1)" | base64 | tr -d '\n')
curl -H "Authorization: Basic $TOKEN" https://mir.advin.io/latest/api/v2.0.0/status
MISSION=$(curl -s -H "Authorization: Basic $TOKEN" https://mir.advin.io/latest/api/v2.0.0/missions | python3 -c 'import json,sys; print(json.load(sys.stdin)[0]["guid"])')
curl -X POST -H "Authorization: Basic $TOKEN" -H 'Content-Type: application/json' \
  -d "{\"mission_id\": \"$MISSION\"}" https://mir.advin.io/latest/api/v2.0.0/mission_queue

      

      

        2 — Or skip the terminal. The console above has the full API catalog,
        generated request bodies, live telemetry, and a curl mirror of every request it sends.

        the repo and
        uv run mir-emulator --mir-version 3.8.1 for a robot on localhost,
        CI, or an air-gapped bench.
      

    

  


  
Documentation & primary sources

  

    Everything this emulator serves is derived from MiR's official REST API documentation — no invented
    endpoints, no guessed schemas. Check its answers against the primary sources yourself:
  

  

    

      
The official MiR API docs

      
Mobile Industrial Robots publishes the REST API reference on its
        support
        portal ("REST API files") — one PDF per product per software version, behind a free account.
        That page is the primary source for every spec used here.

    

    

      
PDF → machine-readable spec

      
MiR ships the API definition as PDFs only. A weekly scraper converts each tracked version's MIR250 PDF
        back to Swagger 2.0, gated by a correctness oracle: the one machine-readable spec MiR ever published
        (3.5.4 swagger.json) must re-convert with zero structural differences on every run.
        See the README
        for the full pipeline.

    

    

      
Verifiable provenance

      
The spec
        registry records, for every tracked version, the source PDF URL and SHA-256 of both the PDF and the
        converted spec. The emulator serves its own definitions at /<version>/swagger.json and
        /<version>/openapi.json, so contract-testing tools can point straight at it.
