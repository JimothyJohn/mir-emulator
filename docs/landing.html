A spec-faithful, open-source emulator of the Mobile Industrial Robots REST API. Every tracked MiR software version is served from this endpoint, so you can develop, harden, and regression-test your fleet integration before it ever touches hardware — then point the same code at a real robot's IP.

Why an emulator

Robot time is scarce and mistakes on real hardware are expensive. This endpoint gives you the API surface with none of the blast radius.

Every tracked version

Each MiR software generation is mounted at its own prefix — /<version>/api/v2.0.0/… — so you can diff behavior across releases from one URL and catch upgrade breakage early.

Real behavior, real auth

Missions execute over simulated time: the queue drains FIFO, battery drains, position moves, PLC registers hold writes. Auth mirrors a factory robot — Basic BASE64(user:SHA-256(password)), default distributor/distributor.

Made for hardening

Deterministic responses generated from the official API definitions, documented error paths (400/404/409) you can trigger on purpose, and per-session virtual robots — send an X-MiR-Session header and you get your own isolated fleet state.

Start in thirty seconds

1 — From your terminal. The demo speaks plain HTTPS; the only trick is MiR's hashed-password Basic auth: 
TOKEN=$(printf 'distributor:%s' "$(printf 'distributor' | shasum -a 256 | cut -d' ' -f1)" | base64 | tr -d '\n')
curl -H "Authorization: Basic $TOKEN" https://mir.advin.io/latest/api/v2.0.0/status
2 — Or skip the terminal. The console has the full 289-operation API catalog, generated request bodies, live telemetry, and a curl mirror of every request it sends.
3 — Run it yourself. The emulator is a Python package: clone the repo and uv run mir-emulator --mir-version 3.8.1 for a robot on localhost, CI, or an air-gapped bench.

MiR Emulator is an independent open-source project; not affiliated with Mobile Industrial Robots A/S. This is a shared demo instance: state is in-memory per runtime instance and resets on cold start — don't store anything you need to keep. Rate-limited and concurrency-capped.
